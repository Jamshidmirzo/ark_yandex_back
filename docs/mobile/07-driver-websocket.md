# 07 — Вебсокет водителя (real-time сокет приложения водителя)

> ЕДИНСТВЕННЫЙ сокет, который нужен приложению водителя. Телефон льёт GPS и на каждый кадр получает,
> куда поставить маркер и какую линию маршрута нарисовать. Все сокеты — в
> [06-websockets.md](06-websockets.md); этапы поездки — [03-scheduling-overlay.md](03-scheduling-overlay.md) §3.6.

```
ws://<host>:8000/ws/driver/track/
```
Двунаправленный. Dev: `ws://<IP-хоста>:8000`; прод: `wss://<домен>`.

---

## 1. Подключение и идентификация
```
ws://<host>:8000/ws/driver/track/?token=<demo JWT>      ← рекомендуется
ws://<host>:8000/ws/driver/track/?driver_id=670         ← dev-фолбэк
```
- Идентификация: **`?token=`** (тот же demo JWT, что и для HTTP — валидируется на сервере) или
  **`?driver_id=`** (только dev). Можно прислать и первым сообщением: `{"token":"…"}` / `{"driver_id":670}`.
- Сразу после connect сервер отвечает один раз:
  ```json
  { "ok": true, "driver_id": 670 }
  ```
  Если `driver_id` = `null` — токен/driver_id не приняты; почини авторизацию до стрима.
- Lang-префикс ок (`/ru/ws/driver/track/`), но лучше без него.

## 2. Отправка — свой GPS (каждые ~5–10 c)
На каждый фикс геолокации шли один кадр:
```json
{ "lat": 41.30050, "lng": 69.20050 }
```
Всё — `order_id` не нужен, сервер сам определит активный заказ.

## 3. Приём — маркер + полилайн (один ответ на кадр)
```json
{
  "order_id": 88,
  "trip_state": "to_client",
  "lat": 41.30050,
  "lng": 69.20050,
  "geometry": [[69.20,41.30], [69.21,41.305], ...]
}
```
| Поле | Что значит | Что делать |
|---|---|---|
| `lat`, `lng` | где маркер сейчас | **двигай маркер** в эту точку (анимируй по линии) |
| `geometry` | полилайн текущего участка, пары `[lng,lat]` — **приходит только когда ИЗМЕНИЛСЯ** | **перерисуй** маршрут; нет поля — оставь прежний |
| `order_id` | текущий активный заказ (`null` = нет, просто на смене) | привяжи UI к этому заказу |
| `trip_state` | этап заказа | обнови баннер статуса |

Когда `geometry` меняется (и значит приходит):
- заказ **назначен** / нажал **«Выехал»** → подъездной маршрут **твоя позиция → подача**;
- **смена этапа** (подача→назначение, обратная нога);
- **отклонение от маршрута** — свернул не туда (отошёл **>80 м** от линии) → сервер перестраивает
  маршрут от текущей точки по новой дороге (как навигатор). Едешь прямо по линии — пересчёта нет.

Итог: **всегда** двигай маркер на `lat/lng`; **заменяй** маршрут только когда пришёл новый `geometry`.

## 4. Реконнект и офлайн
- На разрыв — переподключайся через ~2 c. Догонять не надо — следующий ответ несёт текущее состояние.
- Нет связи → **копи** фиксы и дошли, когда сокет восстановится (не теряй GPS).

## 5. Пример Flutter
```dart
final ws = WebSocketChannel.connect(
  Uri.parse('ws://$host:8000/ws/driver/track/?token=$jwt'));

ws.stream.listen((raw) {
  final m = jsonDecode(raw) as Map;
  if (m['ok'] == true) { /* опознан водитель m['driver_id'] */ }
  if (m['lat'] != null) moveMarker(m['lat'] * 1.0, m['lng'] * 1.0);   // маркер по линии
  if (m['geometry'] != null) {                                         // новый полилайн → перерисовать
    final pts = (m['geometry'] as List)
        .map((p) => LatLng((p[1] as num).toDouble(), (p[0] as num).toDouble())) // [lng,lat] → LatLng
        .toList();
    drawRoute(pts);
  }
  currentOrderId = m['order_id'];
  currentStage   = m['trip_state'];
}, onDone: reconnectAfter2s, onError: (_) => reconnectAfter2s());

// на каждый фикс GPS:
ws.sink.add(jsonEncode({'lat': pos.latitude, 'lng': pos.longitude}));
```

## 6. Чего этот сокет НЕ делает
- **Не двигает этапы.** «Выехал / на месте / начать / завершить» — это отдельный
  `POST /car-orders/{id}/trip-state/` (см. [03 §3.6](03-scheduling-overlay.md)). После POST'а этапа
  сокет начнёт слать `geometry` уже новой ноги.
- **Это не вид заказчика.** Заказчик/диспетчер смотрит за машиной на `ws/order/{order_id}/track/`
  ([06 §2](06-websockets.md)); сервер связывает вас по заказу.
- **Это не единственный способ слать GPS.** HTTP `POST /drivers/me/location/` делает то же самое
  ([04 §4.2](04-live-tracking.md)) — выбери сокет ИЛИ HTTP, не оба.

## 7. Чек-лист
1. Вышел на смену (`PATCH /drivers/me/shift/`) → открой `ws/driver/track/?token=`.
2. Шли `{lat,lng}` каждые 5–10 c; двигай маркер на `lat/lng` из ответа.
3. Пришёл `geometry` — перерисуй линию маршрута.
4. Веди этапы через `POST /trip-state/`; сокет подхватит нужную ногу.
5. Реконнект на разрыв; копи GPS офлайн.
