# Раздел «📋 Заявки V2» — анализ дизайна (ARK 2.0)

> Источник: Figma — Ark 2.0 New Design, страница `📋Заявки _V2⭕️` (node `2567:47308`).
> Полотно ~35 000 × 40 680 px содержит **12 секций**: рабочие макеты (desktop + mobile),
> старые версии (OLD), концепты, библиотеку компонентов и референсы.

Раздел описывает модуль **«Заявки»** — управление заявками на технику/съёмку,
водителями и автопарком (гаражом). Реализован в двух платформах:
**веб-панель (desktop)** для координатора/администратора и **мобильное приложение**
для водителя и координатора.

## 2. Веб-панель (Desktop)

### 2.1. Глобальный каркас (Layout)
Все desktop-экраны построены по единой сетке:

- **Левый сайдбар (Sidebar New, ширина ~250px)** — логотип ARK + меню:
  - `Главная`
  - `Проекты и задачи`
  - `Кадры / Отдел кадров`
  - `Документы`
  - `Система наблюдения`
  - **`Заявки`** (раскрывающийся пункт) →
    - `Заявки на съёмку`
    - `Заявки на машину`
    - `Водители`
    - `Гараж`
  - `Чат`
  - `Календарь`
  - внизу: `Жалоба`, `Выйти`
- **Верхняя панель (Top bar)** — заголовок экрана + хлебные крошки (breadcrumbs),
  справа: поиск 🔍, уведомления 🔔 (с индикатором), переключатель языка `РУ`,
  аватар пользователя с выпадающим меню.
- **Контентная область** — таблица или сетка карточек.

### 2.2. Экран «Водители» (список)
Назначение: каталог водителей компании.

- Заголовок **«Водители»**, хлебные крошки.
- Фильтр по статусам сверху + поле **«Поиск сотрудника»**.
- Кнопка **`+ Новый водитель`** (синяя, справа).
- **Сетка карточек** водителя: аватар, ФИО (Дмитрий Сидоров),
  гос. номер авто (`Damas 154VGN`), статус доступности
  (бейдж **«Доступен» / «Не доступен»**).
- Внизу — **пагинация** (1 · 2 · 3 …).

### 2.3. Экран «Профиль водителя»
Назначение: карточка конкретного водителя и история его заявок.

- Шапка профиля: аватар, **ФИО**, телефон, бейдж статуса (`Доступен`).
- Действия справа: **`Назначить авто`**, **`Удалить`** (красная).
- Блок **«История»** — таблица заявок, сгруппированная по датам (11.12.2025 и т.д.):
  - колонки: **Адрес** · **Статус** · **Авто** · **Время** · меню действий `…`
  - чекбоксы для множественного выбора строк.
  - статусы строк: **Завершено**, **Отменён**, **Принял** (цветные бейджи).
  - кнопка **`Отчет +`** для выгрузки/прикрепления отчёта (рядом — превью
    прикреплённого файла, напр. `07_FxLDACG.large.jpeg`).

### 2.4. Экран «Заявки на машину» (список)
Назначение: реестр заявок на технику.

- Заголовок **«Заявки на машину»**, подпись «Все заявки».
- Поле **«Поиск документа»** + кнопка **`+ Новая заявка`**.
- **Таблица заявок**:
  - колонки: **Тип** (Damas / Faw / Bmw) · **Статус** · **Создал** (аватар) ·
    **Дата создание** · меню `…`
  - сортировка по столбцам (иконки `≡`), чекбоксы выбора строк.
  - статусы заявки: **Новая заявка**, **Отклонено**, **Принято**, **Завершено**.

### 2.5. Экран «Заявка на машину» (карточка/создание)
Назначение: детальная заявка на авто с привязкой к объекту и сотрудникам.

- Хлебные крошки, заголовок заявки.
- Привязка к объекту/проекту (напр. **«Turandot Residences»**).
- Список **«Добавить сотрудника»** (назначение участников на заявку).
- Поля документа, загрузка вложений (Download / Upload), даты (Date Picker).

### 2.6. «Заявки на съёмку»
Отдельный тип заявок (видеосъёмка/контроль объекта) — тот же табличный паттерн,
что и «Заявки на машину», но со своим набором полей и статусов.

### 2.7. «Гараж»
Управление автопарком — список машин (марка, гос. номер, состояние/занятость),
источник данных для назначения авто водителю в профиле.

## 3. Мобильное приложение

### 3.1. Каркас мобильных экранов
- **Status Bar** (системная строка 9:41) + **Top bar** (стрелка «назад»,
  заголовок, действия `+` и поиск 🔍).
- Нижняя навигация (**Bottom Tab**) — в секции `Components` присутствуют
  `Content Container` + `Bottom Tab` (главные вкладки приложения).

### 3.2. Экран «Водители» (mobile)
- Заголовок «Водители», кнопки `+` (добавить) и 🔍 (поиск).
- **Табы:** `Онлайн` / `Оффлайн`.
- **Список карточек** водителя: аватар, ФИО, авто (`Damas 154VGN`),
  индикатор присутствия **Online ●(зелёный) / Offline ●(красный)**,
  бейдж **«Доступен» / «Не доступен»**.

### 3.3. Экран «Детали заявки» (mobile)
- Шапка: ФИО водителя, авто (`Damas A200AB`), телефон, статус.
- **Метрики:** `5 заявок`, `42 Километра` (счётчики по водителю/маршруту).
- Блок **«История»** по датам с пунктами заявок, действия **`Отчет`** / **`Удалить`**.
- **Signer Selection** — выбор подписанта/исполнителя.
- **Modal Date Picker** — выбор даты.
- Карта/локации (`location`) — точки маршрута.

### 3.4. Экран «Профиль водителя» (mobile)
- Та же шапка (ФИО, авто, телефон, статус) + метрики (`5 заявок`, `42 Километра`).
- **История** поездок/заявок по датам с привязанным авто и отчётами.

### 3.5. Экран «Создание водителя» (mobile)
- Форма создания: поля ввода, выбор из списка сотрудников
  (`User_Card-horizontal` — горизонтальные карточки выбора).
- **Signer Selection** + **Date Picker** для назначения.

### 3.6. Чат (Concept mob)
- **Экран чата** между координатором и водителем (переписка по заявке).
- **Toast**-уведомления.
- Экраны `filtr` — фильтрация списков заявок/водителей.

## 4. Роли и сценарии

| Роль | Что делает | Где |
|---|---|---|
| **Водитель** | Видит свои заявки, статус «Онлайн», маршрут, километраж, отправляет отчёт | Mobile |
| **Координатор** | Создаёт заявки на машину/съёмку, назначает водителей и авто, ведёт историю, общается в чате | Desktop + Mobile |
| **Администратор** | Полный доступ к водителям, гаражу, заявкам, управлению | Desktop |

**Ключевой жизненный цикл заявки:**
`Новая заявка → Принято → (выполнение, отчёт) → Завершено`
либо `Отклонено`. Статусы отражаются цветными бейджами одинаково на web и mobile.

## 5. Библиотека компонентов (секция `Components`)

Переиспользуемые элементы раздела:

- **Sidebar New** — левое меню веб-панели.
- **Top bar / Status Bar** — шапки web и mobile.
- **Table** — таблица: `Header`, `Table Content`, `bottom` (пагинация),
  `Table / Searcher`, сортировка `≡`, чекбоксы.
- **Status item / бейджи статусов** — Доступен/Не доступен, Новая/Принято/Завершено/Отклонено.
- **User_Card-horizontal** — карточка сотрудника/водителя (выбор и списки).
- **Input / Tags Input / Modal Date Picker** — поля форм и выбор даты.
- **Bottom Tab + Content Container** — мобильная навигация.
- **Кнопки** (`Button`), иконки (`Download Icon`, `Cancel Icon`, `edit` и др.).
